🔷 Create App in Django (Step-by-Step)
🔶 What is an App?

In Django:

📌 App = A small module that performs a specific function

Examples:
Login system
Blog system
Student management

👉 A project can contain multiple apps

🔷 Step 1: Go to Project Folder

Make sure you are inside your project directory (where manage.py is present):

cd myproject

🔷 Step 2: Create App
📌 Command:
python manage.py startapp myapp


👉 You can replace myapp with any name:

blog
students
accounts
🔷 Step 3: App Structure (Very Important)

After creating app, folder will look like:

myapp/
│
├── __init__.py
├── admin.py
├── apps.py
├── models.py
├── views.py
├── tests.py
└── migrations/

🔶 File Explanation
🔸 models.py

👉 Used to create database tables

🔸 views.py

👉 Contains logic (functions)

🔸 admin.py

👉 Register models for admin panel

🔸 apps.py

👉 App configuration

🔸 migrations/

👉 Stores database changes

🔷 Step 4: Register App (VERY IMPORTANT)

Go to:

myproject/settings.py


Find:

INSTALLED_APPS = [


Add your app:

'myapp',


👉 Example:

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'myapp',
]

🔷 Step 5: Create First View

👉 Open views.py:

from django.http import HttpResponse

def home(request):
    return HttpResponse("Welcome to My App 🚀")

🔷 Step 6: Connect App with URL

👉 Open myproject/urls.py:

from django.contrib import admin
from django.urls import path
from myapp import views

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', views.home),
]

🔷 Step 7: Run Server
python manage.py runserver


👉 Open browser:

http://127.0.0.1:8000/


✅ Output:

Welcome to My App 🚀

🔷 Important Concept (Exam Point)
🔁 Flow:
User → URL → View → Response

🔷 Common Mistakes (Students Make)

❌ Forget to add app in INSTALLED_APPS
❌ Running command outside project folder
❌ Typo in app name

🔷 Pro Tip

👉 Use meaningful app names:

Purpose	App Name
Login system	accounts
Blog	blog
Students	students
🔷 Final Summary
App = module
Created using startapp
Must be registered
Contains models, views, admin
